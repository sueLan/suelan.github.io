XMPP注册流程

1. client发送消息体, 去服务端查询注册需要的字段

xmlns是 jabber:iq:register, type是get

2.1. 未注册：返回注册需要的字段

<instructions/> element：SHOULD contain an element (whose XML character data MAY be modified to reflect the fact that the entity is currently registered)

2.2. 已注册：服务端的返回结果

host会根据”from”的地址判断entity是否已经注册了，IQ result消息有一个空的<registered/>， 标示该entiry已经注册过了。

2.2.Error Case

Condition Description
<bad-request/> The element was not the only child element of the element.
<forbidden/> 权限不够
<not-allowed/> 不允许用户注销账户
<registration-required/> 要注销的账户本来就不存在
<unexpected-request/> The host is an instant messaging server and the IQ get does not contain a ‘from’ address because the entity is not registered with the server.
